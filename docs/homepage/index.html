Web请求的全部流程[仅后端部分]#

浏览器客户端部分#
  1. 在浏览器内按下回车按钮,浏览器程序做为客户端(Client)向浏览器内输入的URL地址发送请求
  2. 从用户输入的URL查找对应的服务器IP 2.1 从操作系统内的host中查找本地的URL 与 IP映射关系,如果找到直接返回 2.2 如果未从host中找到,发请求到运营商DNS服务器查找,如果找不到就去根域名服务器查找,找到之后返回服务器IP地址
  3. 浏览器打包HTTP/HTTPS协议内容到TCP数据包内encode为二进制格式传送给操作系统内核
操作系统(内核)部分#
  1. 操作系统内核在获得浏览器传入的数据后通过网卡对数据进行编码,转换为二进制
  2. 操作系统根据服务器IP地址与对方进行连接
    • 4.1 通过MAC地址,IP地址,路由器等查找
    • 4.2 按照常用的IP协议进行数据传输
  3. 找到服务器后向服务器发送TCP验证请求(三次握手),握手成功后进行数据发送,操作系统内核将TCP数据包进行分包通过IP协议,物理传输(电信号等等)等协议发送到对方服务器
硬件负载均衡部分#
服务器端操作系统部分#
  1. 服务器端计算机操作系统 在与客户端建立的连接中 接收客户端传送过来的电信号
  2. 通过网卡将电信号转换为二进制并且解码为可读信息(十六进制,ASCII等等)
  3. 数据解码后从中提取端口,并在操作系统中查找使用该端口的程序并转发数据到该程序
WebServer(Web应用程序服务员)#

[假设操作系统中注册端口的程序为Nginx] 10. Nginx在master主进程中通过epoll 多路复用技术接收来自操作系统内核的数据,并将数据存储在buffer中缓冲不同设备速度的差异 11. 如果Nginx的配置文件中有upstream分流配置将请求转发到其他地方 12. Nginx将请求根据算法分配到worker进程中 13. Worker进程根据Nginx配置文件进行数据处理,如果是PHP文件,通过fastcgi协议将数据转发到PHP的php-fpm主进程

PHP部分#
  1. php-fpm master主进程根据配置和算法分配请求到worker进程处理
  2. worker进程将请求转发到php的 fpm SAPI
  3. php解释器在收到请求后进行了如下步骤操作 16.1 词法分析(提取token) 16.2 语法分析(提取AST语法树) 16.3 读取可执行码缓存,如果可用直接在此处执行 16.4 生成可执行码 16.3.1 生成Zend VM Opcode 16.3.2 通过JIT即时编译技术生成汇编语言,二进制程序 16.5 缓存生成的可执行码 16.6 如果是zend opcode 则在ZEND VM(虚拟机)上运行程序,如果是汇编程序直接走CPU直接执行
  4. phpfpm worker从解释器中拿到输出结果后返回给Nginx,并通过计算机网络返回给浏览器展示
